Standard Client Endpoints

Lookup Single Number

Retrieves information for a given number, utilizing the client's custom cache TTL and daily/rate limits.

GET POST /

Parameters

  • number (required): The number to look up.
  • key (required): Your API access key.
  • fields (optional): Comma-separated list of fields to return (e.g., name,city,status).

Example Request (GET)

{{ base_url }}/?number=1234567890&key={{ example_key }}&fields=status,name

Example Response (Success)

{
    "success": true,
    "number": "1234567890",
    "source": "cache", 
    "cache_ttl_seconds": 3600,
    "results": {
        "status": "active",
        "name": "John Doe"
    }
}

‚ö° Bulk Lookup

Processes a list of numbers in a single request. Each number counts as **one** hit against your rate limit.

POST /bulk_lookup

Parameters

  • numbers (required): JSON array of numbers to look up (Max 50 items).
  • key (required): Your API access key (in query or header).

Example Request (POST JSON Body)

{
    "numbers": ["1234567890", "9876543210"],
    "key": "{{ example_key }}" 
}

Example Response (Mixed Results)

{
    "success": true,
    "total_requested": 2,
    "results": [
        {
            "number": "1234567890",
            "status": "success",
            "source": "external_api",
            "results": {"status": "active"}
        },
        {
            "number": "0000000000",
            "status": "failure",
            "error": "Number not found or invalid format"
        }
    ]
}

üëë Administration Endpoints

All Admin Endpoints require the X-Admin-Key header to be set to the **Master Admin Key** ({{ admin_key }}).

Manage Keys (Create/Delete)

Create client API keys and assign them a specific tier and cache TTL.

POST /admin/add_key

Parameters

  • key (required): The new API key string.
  • user_id (required): The user/client name.
  • tier_id (required): The subscription level (e.g., basic, pro).
  • ttl (optional): Cache TTL in seconds.

Example Request (Add)

{{ base_url }}/admin/add_key?key=NEW_KEY&user_id=ClientZ&tier_id=basic&ttl=3600
